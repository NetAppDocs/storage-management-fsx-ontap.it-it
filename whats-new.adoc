== 06 ottobre 2025



=== BlueXP è ora NetApp Console

NetApp Console, basata sulle fondamenta BlueXP migliorate e ristrutturate, offre una gestione centralizzata dello storage NetApp e NetApp Data Services in ambienti on-premise e cloud di livello aziendale, offrendo informazioni in tempo reale, flussi di lavoro più rapidi e un'amministrazione semplificata, il tutto in modo altamente sicuro e conforme.

Per i dettagli su cosa è cambiato, vederelink:https://docs.netapp.com/us-en/bluexp-relnotes/index.html["Note sulla versione NetApp Console ."^]

=== Implementare le best practice per un file system FSx per ONTAP

BlueXP Workloads fornisce una dashboard in cui è possibile esaminare lo stato ben strutturato delle configurazioni del file system.  È possibile sfruttare questa analisi per implementare le best practice per i file system FSx for ONTAP .  L'analisi della configurazione del file system include le seguenti configurazioni: soglia di capacità SSD, snapshot locali pianificati, backup FSx per ONTAP pianificati, suddivisione in livelli dei dati e replica remota dei dati.

* link:https://docs.netapp.com/us-en/workload-fsx-ontap/configuration-analysis.html["Scopri l'analisi ben progettata per le configurazioni del file system"]
* link:https://review.docs.netapp.com/us-en/workload-fsx-ontap_well-architected/improve-configurations.html["Implementa le migliori pratiche per i tuoi file system"]

Nella console BlueXP , i file system FSx per ONTAP con problemi ben progettati ora visualizzano una notifica in Canvas che indica quando i file system presentano problemi da risolvere.

== 02 febbraio 2025



=== Associare il file system FSx for ONTAP a uno spazio di lavoro in BlueXP

Dopo l'integrazione BlueXP nel novembre 2024, i file system FSx per ONTAP appena creati non erano associati a un'area di lavoro in BlueXP.  Ora, quando si creano o si scoprono file system FSx per ONTAP , questi vengono associati a un'area di lavoro all'interno di un account BlueXP .

Se disponi di file system FSx per ONTAP esistenti che non sono associati a un'area di lavoro, ti aiuteremo ad associarli a un'area di lavoro in BlueXP.  Puoilink:https://docs.netapp.com/us-en/bluexp-setup-admin/task-get-help.html#create-a-case-with-netapp-support["crea un caso con il supporto NetApp"^] dall'interno della console BlueXP .  Selezionare *Workload Factory* come servizio.



=== Rimozione del file system dalla tela BlueXP

Ora è possibile rimuovere un file system FSx for ONTAP da un'area di lavoro nell'area di lavoro BlueXP .  Questa operazione dissocia il file system da un'area di lavoro in modo da poterlo associare a un'altra area di lavoro all'interno dello stesso account BlueXP .

link:https://docs.netapp.com/us-en/bluexp-fsx-ontap/use/task-remove-filesystem.html["Scopri come rimuovere un file system FSx for ONTAP da un'area di lavoro in BlueXP"^]



=== Tracker disponibile per il monitoraggio e il tracciamento delle operazioni

Tracker, una nuova funzionalità di monitoraggio, è disponibile in BlueXP Amazon FSx for NetApp ONTAP.  È possibile utilizzare Tracker per monitorare e tenere traccia dell'avanzamento e dello stato delle operazioni di credenziali, archiviazione e collegamento, rivedere i dettagli delle attività e sottoattività delle operazioni, diagnosticare eventuali problemi o errori, modificare i parametri per le operazioni non riuscite e riprovare le operazioni non riuscite.

link:https://docs.netapp.com/us-en/bluexp-fsx-ontap/use/task-monitor-operations.html["Scopri come monitorare e tracciare le operazioni FSx per ONTAP in BlueXP"^]



=== CloudShell disponibile nei carichi di lavoro BlueXP

CloudShell è disponibile quando ci si trova in BlueXP Workloads all'interno della console BlueXP .  CloudShell ti consente di utilizzare le credenziali AWS e ONTAP che hai fornito nel tuo account BlueXP ed eseguire comandi AWS CLI o comandi ONTAP CLI in un ambiente simile a una shell.

link:https://docs.netapp.com/us-en/workload-setup-admin/use-cloudshell.html["Utilizzare CloudShell"^]



== 06 gennaio 2025



=== NetApp rilascia ulteriori risorse CloudFormation

NetApp ora fornisce risorse CloudFormation che consentono ai clienti di utilizzare componenti ONTAP avanzati che non sono esposti nella console AWS.  CloudFormation è il meccanismo di infrastruttura come codice per AWS.  Potrai creare relazioni di replicazione, condivisioni CIFS, policy di esportazione NFS, snapshot e molto altro.

link:https://docs.netapp.com/us-en/bluexp-fsx-ontap/use/task-manage-working-environment.html["Gestisci i file system Amazon FSx for NetApp ONTAP utilizzando CloudFormation"]



== 11 novembre 2024



=== FSx per ONTAP si integra con Storage in BlueXP Workload Factory

Le attività di gestione del file system FSx for ONTAP , come l'aggiunta di volumi, l'espansione della capacità del file system e la gestione delle VM di storage, sono ora gestite in BlueXP workload factory, un nuovo servizio offerto da NetApp e Amazon FSx for NetApp ONTAP.  Puoi utilizzare le tue credenziali e autorizzazioni esistenti proprio come prima.  La differenza è che ora puoi fare di più con BlueXP workload factory per gestire i tuoi file system.  Quando si apre un ambiente di lavoro FSx per ONTAP dal canvas BlueXP , si accede direttamente alla BlueXP workload factory.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/learn-fsx-ontap.html#features["Scopri le funzionalità di FSx per ONTAP nella BlueXP workload factory"^]

Se stai cercando l'opzione _visualizzazione avanzata_, che ti consente di gestire un file system FSx per ONTAP utilizzando ONTAP System Manager, ora puoi trovare tale opzione nell'area di lavoro BlueXP dopo aver selezionato l'ambiente di lavoro.

image:https://raw.githubusercontent.com/NetAppDocs/bluexp-fsx-ontap/main/media/screenshot-system-manager.png["Uno screenshot del pannello destro su BlueXP Canvas dopo aver selezionato un ambiente di lavoro che mostra l'opzione System Manager."]
